# Coder Workflow

1. Read `AGENTS.md`, `CURRENT_STATE.md`, and documents relevant to the task.
2. Treat one task as one stage or explicitly assigned substage.
3. Confirm the requested scope; do not add adjacent features.
4. Make small, reviewable changes that preserve existing behavior.
5. Update documentation and tests when their assumptions change.
6. Run build, lint, and relevant runtime checks.
7. Report changed files, implementation, checks, verdict, commit, and risks.
8. Commit only after successful verification.
9. Stop at the assigned stage and wait for approval.
